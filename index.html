📌 Progress 1 · Replication & Minute-Level Pipeline Validation

As of April 14, 2025, we successfully replicated the AI-Trader hourly baseline and built the full 10-minute data pipeline → features → environment → simple strategy flow, proving technical feasibility for high-frequency optimization.

Environment replication & baseline ✅ Done
Fully ran the original project (hourly BTC/USDT), aligned with six major LLM evaluations, logged dependencies, and produced reproducible configs (requirements.txt + Dockerfile). Hourly Sharpe ratio deviation <5% compared to the original report.
10-Minute Data Pipeline ✅ Done
Built minute-level K-line fetching using ccxt + exchange API, storing OHLCV data at 10-minute granularity (Parquet format) covering Q1–Q2 2024. Full data cleaning and anomaly handling, with ingestion efficiency >1000 rows/0.2s.
Minute-level Feature Engineering ✅ Done
Implemented high-frequency indicators: RSI(14), MACD(12,26,9), Bollinger Bands(20,2), ATR(14). Features aligned to 10-minute timestamps with hourly counterparts for comparison. Feature extraction module integrated into a feature factory.
Trading Environment Refactor + Cost Model ✅ Done
Environment time step refactored to 10 minutes, dynamic slippage model (based on historical volatility) and two-way transaction fee (0.075%) introduced. Reward function incorporates profit – risk penalty (high-frequency costs significantly impact net return). Simple moving average strategy runs smoothly.
Initial Comparative Experiment (Rule-based) ✅ Done
Used dual moving average crossover (5/20 periods) backtested on both hourly and 10-minute data. Metrics below show faster signal response potential for the 10-minute pipeline, but transaction costs become significant — next step is LLM-driven optimization.

⚠️ Risk Management & Mitigation

Computational resource explosion – 10‑minute data is 6× larger; we adopted parquet columnar storage + incremental backtesting and plan to use cloud GPU instances.
LLM decision latency – Lightweight inference (vLLM) and asynchronous agent design; if latency exceeds 1s we fall back to rule‑based strategies.
Overfitting risk – Strict train/validation/test time splits and rolling window evaluation.
